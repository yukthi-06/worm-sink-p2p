# Walkthrough - Fix Already Completed Transfer Hang

We have resolved the hang/deadlock when starting a transfer that is already 100% completed.

## Changes Made

### `TransferEngine.java`
- Modified the receiver's `handleProtocolFrame(METADATA)` case to immediately check `if (transferState.completedChunks.size() == transferState.totalChunks)`.
- If true (meaning the file is already 100% downloaded):
  - Calculates the SHA-256 integrity hash of the completed file immediately.
  - Deletes the state file.
  - Sends a `COMPLETE` frame back to the sender.
  - Releases both the connection and session latches (`releaseLatches()`) to exit cleanly.

## Verification
- Restarting a 100% completed download immediately performs the hash verification, notifies the sender to close, and exits successfully instead of hanging.
