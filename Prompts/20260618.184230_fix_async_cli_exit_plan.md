# Goal Description

Fix the `VM attach current thread failed` / `Failed to attach thread` JNI warnings at successful completion or failure. 

Because `Runtime.getRuntime().halt()` is called directly inside the WebRTC JNI callback thread, the JVM is terminated while the native thread is still executing Java code. This causes JNI/VM structures to go out of sync during destruction.

We will run the exit logic (`Runtime.getRuntime().halt()`) asynchronously on a separate Java thread after a 200 ms delay. This allows the JNI thread to return cleanly to C++ and detach from the VM before the process is terminated.
