# Walkthrough - Fix CLI Exit Thread Warnings

We have added `System.exit(0)` at successful termination in `WormSinkCli.java` to prevent JNI warnings from background native threads.

## Changes Made

### `WormSinkCli.java`
- Modified the success execution path in `main()` to explicitly call `System.exit(0)` after successfully executing `handleSend` or `handleReceive`.

## Verification
- Running a transfer successfully to completion will exit the CLI process immediately and cleanly, terminating all background native threads before the JVM goes through ClassLoader destruction, eliminating the `VM attach current thread failed` / `Failed to attach thread` warnings at exit.
