# Walkthrough - Avoid Re-downloading Finished Files

We have implemented an early-completion check for finished files on disk to prevent overwriting and re-downloading them.

## Changes Made

### `TransferEngine.java`
- In the receiver's `handleProtocolFrame(METADATA)` case, added a check before loading/initializing the `.state` file:
  - If a file exists at `destinationPath` and matches the expected `fileSize`:
    - Calculate the SHA-256 hash of the local file.
    - If the hash matches `fileHash` from the metadata:
      - Call `onTransferStarted()` (so the CLI displays the metadata).
      - Set `bytesTransferred` to `fileSize`.
      - Set `transferCompleted = true`.
      - Delete the `.state` file (if any exists).
      - Send a `COMPLETE` frame back to the sender.
      - Release latches to exit cleanly without performing any network chunk downloads.

## Verification
- Starting the download of an already fully downloaded file immediately verifies its hash, signals completion, prints `Download completed successfully!`, and exits without truncating or re-downloading the file.
