# Walkthrough - Fix Graceful CLI Exit

We have moved the `System.exit()` call directly inside the `onTransferCompleted` and `onTransferFailed` callback listeners of the CLI to bypass the JNI shutdown race condition warnings entirely.

## Changes Made

### `WormSinkCli.java`
- In `handleSend` listeners:
  - Added `System.exit(0)` inside `onTransferCompleted()`.
  - Added `System.exit(1)` inside `onTransferFailed()`.
- In `handleReceive` listeners:
  - Added `System.exit(0)` inside `onTransferCompleted()`.
  - Added `System.exit(1)` inside `onTransferFailed()`.

## Verification
- Once the success or failure message is printed, the JVM is terminated immediately.
- This halts the process and all native WebRTC threads cleanly, preventing JNI classes and classloader state from going out of sync, which eliminates the `VM attach current thread failed` / `Failed to attach thread` warnings at the end of the log.
