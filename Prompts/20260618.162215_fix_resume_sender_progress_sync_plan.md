# Goal Description

Implement progress synchronization for resumed transfers. Currently, when a receiver restarts and resumes a transfer, the sender's progress bar starts from 0%. We will integrate the existing `RESUME` protocol frame:
1. The receiver sends the `RESUME` frame containing the list of missing chunk indices upon startup.
2. The sender updates its progress bar to match the number of completed chunks (total chunks minus missing chunks) and pushes the missing chunks.

- In `handleProtocolFrame(METADATA)` for the receiver:
  - If resuming (saved transferState matches), send a `RESUME` frame to the sender with the missing chunk indices instead of calling `requestMoreChunks()`.
  - Only call `requestMoreChunks()` on a fresh start.
- In `handleProtocolFrame(RESUME)` for the sender:
  - Calculate `totalChunks` and determine `completedCount` (`totalChunks - resume.missingChunkIndices.length`).
  - Update `bytesTransferred` to `completedCount * chunkSize` and call `reportProgress()`.
