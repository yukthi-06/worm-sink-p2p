# Walkthrough - Synchronize Resumed Sender Progress

We have integrated the `RESUME` frame to synchronize progress bars on both the sender and receiver sides during resumed transfers.

## Changes Made

### `TransferEngine.java`
- **Receiver METADATA Handling**: Modified the receiver's METADATA handler. If resuming, it now builds a `RESUME` frame containing the missing chunk indices and sends it to the sender. If it's a fresh start, it falls back to calling `requestMoreChunks()`.
- **Sender RESUME Handling**: Modified the sender's `RESUME` case. When the sender receives a `RESUME` frame, it calculates the completed chunk count (`totalChunks - missingChunkIndices.length`), updates its `bytesTransferred` atomic field, calls `reportProgress()` to sync its console output/progress bar, and begins pushing chunks via `sendChunksAsync()`.

## Verification
- Both the sender and receiver will show the same completed progress (e.g. 30%) immediately upon connection resume.
- The transfer resumes correctly from the exact point of disconnection.
