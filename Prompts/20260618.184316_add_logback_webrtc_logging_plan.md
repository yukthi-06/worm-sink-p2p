## Initial Analysis
We want to:
1. Add Logback and SLF4J dependencies to the multi-module project.
2. Direct all java.util.logging (JUL) outputs to SLF4J.
3. Hook into the `dev.onvoid.webrtc.logging.Logging` class by registering a custom `LogSink` to capture and route native WebRTC logs to SLF4J.
4. Configure Logback via a `logback.xml` file located in the resources directory of `wormsink-cli` to log everything (including SLF4J, JUL, and WebRTC native logs) to a file (`wormsink.log`) and prevent them from being output to the console.
