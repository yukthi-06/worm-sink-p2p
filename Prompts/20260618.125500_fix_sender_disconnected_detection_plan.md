# Goal Description

Fix the stale offer deadlock when restarting the receiver. Currently, the sender takes too long (8 seconds via heartbeat watchdog, or waiting for ICE to transition to `FAILED`) to detect that the receiver has disconnected. If the receiver is restarted during this window, it retrieves the stale offer and gets stuck.

We will update the sender to immediately trigger a reconnect/reset when the connection state transitions to `DISCONNECTED`, which WebRTC detects almost immediately.

## Proposed Changes

### [TransferEngine]

#### [MODIFY] [TransferEngine.java](file:///x:/Projects_X/0_Active/1_Java_Active/AI_Assisted/WormSink_Java_GITHUB/wormsink-transfer/src/main/java/org/wormsink/transfer/TransferEngine.java)

- Modify `onConnectionStateChange` to treat `DISCONNECTED` on the sender side the same as `FAILED`/`CLOSED`.
- This will immediately trigger the async `resetSession()` and count down `sessionLatch` to start the reconnect loop.
