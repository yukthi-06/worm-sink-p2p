# Walkthrough - Fix Metadata Unquoted JSON Parsing

We have fixed the simple JSON utility class to support parsing unquoted numeric fields like `fileSize` and `chunkSize` which were failing to match previously.

## Changes Made

1.  **[SimpleJson.java](file:///x:/Projects_X/0_Active/1_Java_Active/AI_Assisted/WormSink_Java_GITHUB/wormsink-signaling/src/main/java/org/wormsink/signaling/SimpleJson.java):**
    - Enhanced the regex inside `getField` from matching only quoted values (`"key":"value"`) to capturing both quoted strings and unquoted numeric/boolean values:
      `Pattern.compile("\"" + field + "\"\\s*:\\s*(?:\"([^\"]*)\"|([^,}\\s]+))")`
    - This successfully avoids `NumberFormatException` when loading `fileSize`, `chunkSize`, and `totalChunks` from the metadata JSON, preventing unexpected application closure.
