# Goal Description

Fix the classloader/JNI thread attachment warnings (`VM attach current thread failed`) at successful CLI exit. When the receiver finishes downloading successfully, the `main()` method exits normally. The JVM then starts class unloading and teardown. However, background native WebRTC threads might still run for a few milliseconds, trying to attach to the JVM to call observers.

We will add a clean `System.exit(0)` call at the end of the `WormSinkCli` `main()` success path to terminate the process and all native threads instantly and cleanly.
