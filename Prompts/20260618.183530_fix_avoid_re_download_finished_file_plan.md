# User Prompt

> once a download is complete and the file is in the destination directory.... When we start the same download again, it creates the *.state file freshly and starts downlading from the first chunk.
> It is supposed to check the file checksum and chunk checksum and determine if download has to be performed or not....

---

# Goal Description

Fix the issue where starting a download for an already fully downloaded file truncates/overwrites the file and starts downloading from the beginning. Since the `.state` file is deleted on successful completion, the receiver assumes it is a fresh transfer.

We will add a check in the receiver's `METADATA` handler: if the destination file already exists, matches the expected size, and matches the expected SHA-256 file hash, we will complete the transfer immediately without downloading any chunks.

## Proposed Changes

### [TransferEngine]

#### [MODIFY] [TransferEngine.java](file:///x:/Projects_X/0_Active/1_Java_Active/AI_Assisted/WormSink_Java_GITHUB/wormsink-transfer/src/main/java/org/wormsink/transfer/TransferEngine.java)

- In the receiver's `handleProtocolFrame(METADATA)` case:
  - Check if `destinationPath` exists as a file and its size matches the metadata `fileSize`.
  - If it matches, calculate the SHA-256 hash of the existing file.
  - If the hash matches `fileHash`, immediately mark `transferCompleted = true`, delete the `.state` file, send a `COMPLETE` frame back to the sender, and release the latches.
