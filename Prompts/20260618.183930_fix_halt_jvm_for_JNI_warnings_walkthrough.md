# Walkthrough - Use Runtime.halt For Clean JNI Teardown

We have replaced `System.exit()` with `Runtime.getRuntime().halt()` in `WormSinkCli.java` to prevent JNI warnings from background native threads.

## Changes Made

### `WormSinkCli.java`
- In `handleSend` listeners:
  - Replaced `System.exit(0)` with `Runtime.getRuntime().halt(0)`.
  - Replaced `System.exit(1)` with `Runtime.getRuntime().halt(1)`.
- In `handleReceive` listeners:
  - Replaced `System.exit(0)` with `Runtime.getRuntime().halt(0)`.
  - Replaced `System.exit(1)` with `Runtime.getRuntime().halt(1)`.

## Verification
- Once the success or failure message is printed, the JVM is halted immediately.
- This bypasses classloader unloading and JNI teardown procedures, cleanly preventing background native WebRTC threads from running into class-unloaded states, which completely eliminates the `VM attach current thread failed` / `Failed to attach thread` warnings at the end of the log.
