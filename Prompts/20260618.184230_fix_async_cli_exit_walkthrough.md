# Walkthrough - Async CLI Exit To Prevent JNI Warnings

We have wrapped the JVM termination code in an asynchronous `CompletableFuture` task with a 200 ms delay to resolve the JNI VM attachment warnings.

## Changes Made

### `WormSinkCli.java`
- Modified both the sender and receiver success and failure callbacks:
  - Wrapped `Runtime.getRuntime().halt(0)` and `Runtime.getRuntime().halt(1)` inside `CompletableFuture.runAsync()` with a `Thread.sleep(200)` delay.

## Verification
- When the download completes, the JNI callback thread finishes printing the completion message, returns cleanly back to the native library, and detaches from the JVM.
- 200 ms later, the asynchronous task forcibly halts the JVM cleanly, preventing any JNI shutdown race condition or thread attachment warnings.
