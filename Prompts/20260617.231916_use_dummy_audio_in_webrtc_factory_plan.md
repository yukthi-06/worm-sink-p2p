## Goal
Resolve connection setup hangs and JNI thread errors when running sender and receiver on the same laptop by configuring the WebRTC `PeerConnectionFactory` to use dummy audio.

## Root Cause Analysis
- By default, WebRTC's `PeerConnectionFactory` initializes the real audio device module (ADM) of the system, trying to access the default microphone and speakers.
- On Windows, audio drivers often restrict or block multiple concurrent processes from initializing/locking the default audio recording device.
- Since we are running the sender and receiver on the same laptop at the same time, this hardware conflict causes the second process (or both) to experience native thread conflicts or hang during initialization.
- Because WormSink is a pure data transfer tool (using DataChannel only), it does not require any real audio devices.
