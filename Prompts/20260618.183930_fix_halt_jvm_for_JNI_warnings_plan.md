# Goal Description

Fix the persistent JNI warning `VM attach current thread failed` / `Failed to attach thread` at successful completion or failure. 

Even with `System.exit(0)`, the JVM performs a clean shutdown, which includes unloading classes and native libraries (DLLs). Because background native WebRTC threads are still cleaning up inside the native library, they attempt to attach to the JVM after JNI classes/DLLs have been unloaded, triggering JNI warnings.

To resolve this, we will use `Runtime.getRuntime().halt(0)` on success and `Runtime.getRuntime().halt(1)` on failure. This forcibly terminates the JVM instantly, bypassing clean class/DLL unloading and preventing native WebRTC threads from running into class-unloaded states.
