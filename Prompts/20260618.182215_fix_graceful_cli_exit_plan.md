# Goal Description

Fix the `VM attach current thread failed` / `Failed to attach thread` JNI warnings printed by the WebRTC library during teardown. 

When the transfer completes or fails, we trigger the cleanup code which calls `connectionManager.close()`. This shuts down the PeerConnection, triggering callbacks on native threads while the JVM is already winding down, leading to JNI thread attachment failures.

To resolve this, we will call `System.exit()` directly inside the CLI's `onTransferCompleted` and `onTransferFailed` listeners. This terminates the process instantly on completion or failure, bypassing the JNI teardown race conditions.
